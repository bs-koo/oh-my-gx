# oh-my-gx

**GX 사업본부의 PM과 개발자가 하나의 흐름으로 일하는 AI 개발 자동화 플러그인**

요청 한 줄이면 PRD 작성부터 커밋/PR 생성까지, AI 에이전트 팀이 전체 개발 사이클을 수행합니다.

</div>

---

## 30초 시작

```bash
# 1. 설치
/plugin marketplace add bs-koo/oh-my-gx
/plugin install oh-my-gx@oh-my-gx

# 2. 초기 설정 (역할 선택 + 환경 구성)
/oh-my-gx:setup

# 3. 첫 작업 시작
/oh-my-gx:dev 상품 목록 정렬 기능 추가
```

## 자연어로 말하세요

플래그를 외울 필요 없습니다. 의도를 말하면 알아서 판단합니다.

| 하고 싶은 말 | 실행되는 기능 | 예시 |
|-------------|-------------|------|
| 기능 추가/수정 | `/dev` 전체 사이클 | "주문 취소 기능 추가" |
| 긴급 수정 | `/dev` hotfix 모드 | "결제 오류 긴급 수정해줘" |
| PRD만 작성 | `/dev` requirements | "PRD만 작성해줘" |
| 이어서/계속 | `/dev` 재개 | "아까 하던 작업 이어서 해줘" |
| 진행 상태 | `/dev` 상태 확인 | "지금 어디까지 됐어?" |
| 팀으로 | `/dev` 팀 모드 | "팀으로 결제 모듈 리팩토링" |
| 도메인 등록 | `/context` | "결제 도메인 등록해줘" |
| 정책 분석 | `/lens` | "구매 한도 정리해줘" |
| 커밋 | `/commit` | "커밋해줘" |
| PR 생성 | `/pull-request` | "PR 만들어줘" |

## 어떻게 작동하나요?

```
자연어 입력 → 의도 파악 → Q&A 인터뷰 → 단계별 실행 → 결과 보고
```

| 단계 | 담당 | 하는 일 |
|------|------|---------|
| 의도 파악 | 오케스트레이터 | 자연어에서 모드/스킬 자동 판정 |
| 인터뷰 | PO(제품책임자) | 요구사항 구체화 질문 |
| 설계 | 설계자 | 기술 설계 + 사용자 승인 |
| 구현 | 개발자 | 코드 작성 + 자기점검 |
| 리뷰 | 리뷰어 | 코드 리뷰 + 보안 감사 |
| 완료 | 오케스트레이터 | 인수 검증 + 커밋/PR |

## 이럴 때 쓰세요

### PM이 기획서를 코드로 만들고 싶을 때

기획 의도를 말하면 PRD를 작성하고, 개발자에게 넘길 수 있는 상태까지 만들어줍니다.

```bash
/oh-my-gx:dev 주문 취소 PRD만 작성해줘
# → AI가 질문으로 요구사항을 구체화 → PRD 문서 생성
```

### 개발자가 반복 작업을 자동화하고 싶을 때

요청 한 줄이면 설계 → 구현 → 리뷰 → 커밋/PR까지 자동으로 진행됩니다.

```bash
/oh-my-gx:dev 결제 오류 긴급 수정해줘
# → 긴급 수정: 설계/리뷰 생략하고 바로 구현 → 커밋 → PR
```

### 코드 변경의 비즈니스 영향을 파악하고 싶을 때

코드를 수정하지 않고 비즈니스 정책만 분석합니다. PM이 개발자 없이 직접 확인할 수 있습니다.

```bash
/oh-my-gx:lens 구매 한도 정리해줘
# → 관련 코드에서 정책을 추출하고 변경 시 영향 범위를 보고
```

## 주요 스킬

### `/dev` — 전체 개발 사이클

요청 한 줄로 PRD → 설계 → 구현 → 리뷰 → 커밋/PR까지 자동 수행합니다.
자연어 트리거는 위의 [자연어로 말하세요](#자연어로-말하세요) 테이블을 참고하세요.

```bash
/oh-my-gx:dev 주문 취소 기능 추가                # 전체 사이클
/oh-my-gx:dev 결제 오류 긴급 수정해줘             # → hotfix 모드 자동 판정
/oh-my-gx:dev 팀으로 결제 모듈 리팩토링           # → 팀 모드 자동 전환
```

| Phase | 하는 일 | 사용자 확인 |
|-------|---------|-----------|
| requirements | PRD 작성 | Q&A + 승인 |
| design | 기술 설계 | Q&A + 승인 |
| implement | 코드 구현 + 자기점검 | 결과 보고 |
| review | 코드 리뷰 + 보안 감사 | 이슈 답변 |
| complete | 인수 검증 + 커밋/PR | 최종 확인 |

**실전 대화 흐름 예시:**

```
사용자: /oh-my-gx:dev 상품 목록에 카테고리 필터 추가

AI(PO): 몇 가지 확인이 필요합니다.
    1. 카테고리는 단일 선택인가요, 복수 선택인가요?
    2. 필터 초기화 버튼이 필요한가요?

사용자: 복수 선택, 초기화 버튼 필요해

AI(PO): PRD를 작성했습니다. 확인해주세요.
    → [PRD 내용 표시]

사용자: 승인

AI(설계자): 기술 설계를 완료했습니다. API 엔드포인트와 DB 스키마를 확인해주세요.
    → [설계 내용 표시]

사용자: 승인

AI(개발자): 구현을 완료했습니다. 변경된 파일 5개, 테스트 통과.
AI(리뷰어): 코드 리뷰 완료. 이슈 없음.
AI: PR을 생성했습니다. → https://github.com/...
```

### `/context` — 도메인 지식 관리

도메인의 용어, 아키텍처, 프로젝트 매핑을 등록하면 `/dev` 시 자동 참조합니다.

```bash
/oh-my-gx:context                              # 코드베이스 스캔으로 자동 생성
/oh-my-gx:context 결제                          # Q&A 기반 새 도메인 추가
/oh-my-gx:context 정산 --from docs/정산-요구사항.md  # 문서 기반 생성 (검증 Q&A 포함)
/oh-my-gx:context 결제 --sync                   # git 히스토리 기반 진행도 동기화
```

### `/commit` — 커밋

브랜치명에서 타입을 파싱하여 한국어 커밋 메시지를 자동 생성합니다.

```bash
/oh-my-gx:commit                # 메시지 자동 생성
/oh-my-gx:commit 로그인 검증 수정  # 메시지 직접 지정
```

### `/pull-request` — PR 생성

커밋 히스토리를 분석하여 PR 제목/본문을 자동 생성합니다. Google Chat 알림이 설정되어 있으면 PR 생성 후 Chat Space에 자동 알림을 전송합니다.

```bash
/oh-my-gx:pull-request          # 베이스 브랜치 자동 감지
/oh-my-gx:pull-request develop  # 베이스 브랜치 지정
```

### `/lens` — 비즈니스 정책 분석

코드에서 비즈니스 정책을 탐지하고 영향도를 분석합니다. 코드를 수정하지 않습니다.

```bash
/oh-my-gx:lens 구매 정책 정리해줘
/oh-my-gx:lens 구매 한도 --idea "한도를 50만원으로 올리면?"
```

### `/humanizer` — AI 글쓰기 교정

AI가 쓴 티를 제거합니다. 감지만 하는 audit 모드와 직접 수정하는 rewrite 모드를 지원합니다.

```bash
/oh-my-gx:humanizer docs/guide.md
```

### `/worktree` — 격리 개발 환경

Git worktree로 격리된 작업 환경을 관리합니다.

```bash
/oh-my-gx:worktree create feat/login
/oh-my-gx:worktree done
```

## 팀 모드

대규모 작업(핵심 파일 4개 이상)이면 AI가 자동으로 팀 모드를 제안합니다.
자연어로 "팀으로 해줘"라고 말하면 확인 없이 바로 전환됩니다.

| 규모 | 핵심 파일 수 | 동작 |
|------|-------------|------|
| 소형 | 3개 이하 | 기본 모드 유지 |
| 중형 | 4~7개 | "팀 모드로 전환할까요?" 제안 |
| 대형 | 8개 이상 | 팀 모드 강력 권장 |

팀 모드에서는 팀장(architect)이 태스크를 분배하고, 복수의 coder가 파일 소유권 기반으로 병렬 구현합니다. 파일 충돌 없이 대규모 변경을 안전하게 처리합니다.

## Google Chat 알림

`/setup` 실행 시 Google Chat 웹훅을 연동하면 PR 생성 시 Chat Space에 자동 알림이 전송됩니다.

| PR 유형 | 알림 메시지 |
|---------|-----------|
| context 파일만 변경 | `[프로젝트명] context가 수정되었습니다. PR을 확인해주세요` |
| 코드 변경 포함 | `[프로젝트명] 기능이 추가/수정되었습니다. PR을 확인해주세요` |

한 명이 setup에서 URL을 설정하고 커밋하면, 다른 팀원은 pull 후 별도 설정 없이 동일한 알림을 받습니다.

## 안전장치

- PR 생성까지만 자동화합니다. **PR 머지는 사용자가 직접** 수행합니다.
- 보호 브랜치(main)에서 직접 커밋을 차단합니다.
- `git push --force`, `gh pr merge`를 차단합니다.
- 커밋 전 민감 파일(`.env` 등) 감지 시 경고합니다.

## FAQ

<details>
<summary><b>context/는 꼭 만들어야 하나요?</b></summary>

아니요. 없어도 동작합니다. 등록하면 에이전트가 도메인 용어와 아키텍처를 정확히 이해하여 더 정확한 코드를 생성합니다.
</details>

<details>
<summary><b>플러그인 업데이트는?</b></summary>

`/plugin marketplace update oh-my-gx`
</details>

<details>
<summary><b>PM도 개발 지식 없이 사용할 수 있나요?</b></summary>

네. `/setup`에서 PM 역할을 선택하면 PRD 작성(`/dev`)과 정책 분석(`/lens`)에 최적화된 환경이 구성됩니다. 코드를 직접 다루지 않아도 됩니다.
</details>

<details>
<summary><b>팀 모드는 언제 쓰나요?</b></summary>

핵심 파일이 4개 이상인 대규모 작업에서 AI가 자동으로 제안합니다. 또는 "팀으로 해줘"라고 직접 요청할 수도 있습니다. 팀 모드에서는 복수 에이전트가 병렬로 작업하여 대규모 변경을 빠르게 처리합니다.
</details>
