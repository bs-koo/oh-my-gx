# 설계 결정 기록

검토 후 채택하지 않은 설계 제안과 그 사유를 기록한다. 같은 제안이 다시 올라올 때 재검토 비용을 줄이는 것이 목적이다.

---

## DD-001: 독립 AC 병렬 RGR (worktree 격리) — 기각 (2026-07-07)

**제안**: gx-tdd의 태스크(AC)별 RGR 사이클을 git worktree로 격리해 병렬 실행하여 처리량을 개선하자.

**기각 사유**:

1. **사이클 단위 회귀 보증 상실 (결정적)**: 현행 verify_green/verify_refactor는 매 GREEN마다 전체 테스트 스위트를 실행해 태스크 간 회귀를 사이클 단위에서 잡는다. worktree로 가르면 각 트리에 다른 태스크의 테스트가 없으므로 교차 회귀가 머지 시점에야 드러난다 — TDD 파이프라인의 핵심 보증이 "사이클마다"에서 "머지할 때 한 번"으로 후퇴한다.
2. **격리 장치 전면 재설계 필요**: Iron Law("쓰기 에이전트 병렬 금지"), 무결성 기준선(`rgr-t{N}-porcelain.txt` 스냅샷, `test-file-hash`, `test-count`), state.md 스키마가 모두 단일 작업 트리를 전제한다.
3. **이득 대비 비용**: 대상 환경(기능당 AC 3~5개, RGR 사이클당 수 분)에서 병렬 이득보다 머지·재검증 오버헤드가 크다. 처리량이 필요한 비-TDD 작업은 gx-dev의 배치 병렬 모드가 이미 담당한다.

**재검토 조건**: 테스트 루트가 완전히 분리된 독립 모듈을 동시 개발하는 요구가 실제로 반복 발생할 때.
